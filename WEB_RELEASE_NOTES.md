# Web Release Notes

## Privacy / Disclaimer Level Needed For This Prototype

For colleague testing, a full legal policy stack is probably not required yet, but these minimum pieces are strongly recommended:

- a visible `reference-only` warning
- a clear note that the app is `not for sole clinical decision-making`
- reference/source context where dosing ranges are shown
- a short privacy note if external feedback forms are used

## Practical Recommendation

Before sharing the public link, it is reasonable to keep the following in place:

- top warning banner
- in-result warning text
- reference notes for infusion and pediatric content
- support tab with feedback link

## Suggested Minimum Privacy Note

If you share this app publicly or semi-publicly, add a short note such as:

`This prototype stores certain preferences locally in your browser, such as language selection, saved favorites, and custom presets. If you use the feedback form, submitted responses are collected through Google Forms.`

## Suggested Minimum Disclaimer Note

`Anestha is a reference-oriented prototype for education and workflow feedback. It must not be used as the sole basis for medication preparation, dosing, or clinical decision-making. Independent verification and institutional protocols remain necessary.`

## When A Dedicated Privacy / Disclaimer Page Becomes Worthwhile

Create separate pages if one or more of these become true:

- the app is shared beyond close colleagues
- support / donation links go live
- analytics are added
- authentication is added
- cloud sync is added
- the app is posted publicly on a stable domain

At that point, separate pages are recommended:
- `Privacy`
- `Disclaimer`
- optionally `Methodology / References`
